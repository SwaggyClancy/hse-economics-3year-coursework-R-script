================================================================================
ТАБЛИЦЫ (output/tables/)
================================================================================

01_chain_stickiness.csv / .html
  Сводная таблица метрик ценовой липкости по каждой сети (Пятёрочка vs Магнит).
  Строк: 2 (по одной на сеть).
  Столбцы: freq_regular (частота изменений регулярной цены), freq_effective
    (частота изменений эффективной цены — то, что платит покупатель),
    avg_change_effective (средний размер изменения, %), promo_share (доля
    акционных периодов), avg_promo_depth (средняя глубина скидки),
    avg_spell_length (средняя длина периода без изменений, нед),
    volatility_effective (стандартное отклонение дельты), n_categories,
    n_products.
  Как читать: большой avg_spell_length = более «липкие» цены.

02_category_stickiness.csv / .html
  То же самое с разбивкой по категориям внутри каждой сети.
  Строк: (кол-во категорий) × (кол-во сетей).
  Позволяет найти категории с аномально высокой/низкой ценовой активностью.

03_cross_chain_correlations.csv / .html
  Корреляция Пирсона еженедельной частоты изменений эффективной цены между
  Пятёрочкой и Магнитом — отдельно для каждой категории.
  Столбцы: category_name, cor_chg_rate (корр. частоты), cor_avg_delta (корр.
    среднего изменения), n_weeks (число общих наблюдений).
  Как читать: cor_chg_rate > 0.5 = высокая синхронность (внешние шоки
    доминируют); cor ≈ 0 = независимые ценовые решения.

04_intercategory_cors.csv
  Попарные корреляции временных рядов chg_rate между категориями внутри
  каждой сети. Строк: C(n_cat,2) × 2 сети.
  Используется для построения тепловых карт и топ-5 корреляций.

04b_intercategory_cors_combined.csv
  То же самое, но chg_rate усреднён по обеим сетям перед расчётом.
  Показывает «общерыночную» синхронность категорий.

05_intercategory_summary.csv
  Сводка межкатегорийных корреляций: mean_cor, median_cor, pct_pos по сети.
  pct_pos = 1.0 у Пятёрочки → централизованное ценообразование;
  pct_pos ≈ 0.5 у Магнита → децентрализованное.

06_variance_decomposition.csv / .html
  Последовательные R² adj. при добавлении FE: только сеть, только категория,
  только магазин, сеть+категория, полная модель. Зависимая: delta_effective_pct.
  Инкрементальный дельтаR² показывает «чистый» вклад каждого уровня.

07_cluster_centroids.csv / .html
  Центроиды K-means кластеров в исходных единицах (не стандартизированных).
  Столбцы: cluster, n_cat, freq_effective, avg_change_effective, promo_share,
    volatility, avg_spell, categories (перечень категорий в кластере).
  K выбирается силуэтом с ограничением MIN_K=3.

08_category_cluster_assignment.csv
  Принадлежность каждой категории к кластеру + все признаки кластеризации.
  Строк: n_categories (объединённые по обеим сетям).

09_regression_results.tex
  Таблица панельных регрессий M1/M2/M3 в формате LaTeX — для вставки в текст
  курсовой. Зависимая: дельта_ln(эффективной цены).

09_regression_coefs.csv
  Коэффициенты M1/M2/M3 в длинном формате: term, estimate, std_error, t_stat,
  p_value, model. Используется для построения coef-plots.

09_regression_coefs.html
  То же, но в виде отформатированной HTML-таблицы с подробными сносками:
  пояснение каждой переменной, FE, кластеризации SE, GoF по спецификациям.

10_regression_gof.csv / .html
  Метрики качества регрессий: N наблюдений, R² within, R² adj.

11_outlier_profile.csv
  Профиль выбросов RC3 (|дельта_ln(P)| > 50%): где концентрируются
  экстремальные изменения? Столбцы: store_chain, cat_group, n_obs,
  avg_abs_chg, max_abs_chg, pct_promo. Помогает понять природу выбросов
  (ошибки парсинга / уценка / ввод товара после перерыва).

11_robustness_coefs.csv / 11_robustness_table.csv / .html
  Коэффициенты бета(Magnit) и бета(Promo) при трёх альтернативных
  спецификациях:
    RC1 — только регулярные цены (без акций);
    RC2 — порог изменения 0.5% вместо 1%;
    RC3 — без наблюдений с |дельта_ln(P)| > 50%.
  Если знак и значимость сохраняются — результаты M1 устойчивы.

12_top5_correlations.csv / .html
  Для каждой категории — топ-5 наиболее коррелирующих категорий (отдельно
  для Пятёрочки, Магнита и их комбинации). Широкий формат.

12_top5_correlations_long.csv
  То же самое в длинном формате с отдельными строками для каждой пары.

13_lpm_coefs.csv
  Коэффициенты LPM-регрессий (3 спецификации). Зависимая: P(изменение
  эффективной цены). бета интерпретируется в процентных пунктах.

13_lpm_gof.csv / .html
  Метрики качества LPM: N, R² within, R² adj.

14_granger_coefs.csv / .html
  Коэффициенты кросс-лаговых регрессий: лаг(Magnit) → Pyaterochka и обратно.
  Значимый бета = Granger-причинность между сетями.

15_ar_coefs.csv
  Коэффициенты AR(1): ро < 0 = «ценовое торможение»; ро > 0 = инерция.
  Три спецификации: базовая / + промо / + промо + сеть.

15_ar_gof.csv / .html
  Метрики качества AR(1).

16_rf_importance_full.csv
  Полный рейтинг важности переменных Random Forest (по каждому уровню FE).

16_rf_importance_grouped.csv
  Агрегированная важность по группам: Категория / Неделя / Магазин / Промо /
  Сеть. Показывает, что доминирует в предсказании изменений цены.

--- СКРИПТ 02: Описательные метрики ---

01_freq_by_category.png
  Тип: bar chart (горизонтальный, dodge по сети).
  Ось X: доля недель с изменением эффективной цены (freq_effective).
  Ось Y: категория товара.
  Две полосы на категорию: Пятёрочка и Магнит (разные цвета).
  Как читать: длинная полоса = цены меняются часто (менее «липкие»).

02_promo_vs_freq.png
  Тип: scatter plot с подписями точек.
  Ось X: доля акционных периодов (promo_share).
  Ось Y: частота изменений эффективной цены (freq_effective).
  Размер точки = количество товаров в категории.
  Как читать: точки в правом верхнем углу = высокая промо-активность И
    высокая частота изменений (акции — главный драйвер изменений).

03_spell_distribution.png
  Тип: гистограмма, facet по сети.
  Ось X: длительность ценового спелла (недель) по ЭФФЕКТИВНОЙ цене.
  Ось Y: количество случаев.
  Ценовой спелл = непрерывный период без изменений эффективной цены.
  Как читать: «хвост» вправо = есть товары с очень долгой стабильностью цен.

04_freq_hist_by_chain.png
  Тип: гистограмма, facet по сети.
  Ось X: частота изменения ЭФФЕКТИВНОЙ цены по товару (freq_effective).
  Ось Y: количество товаров.
  Как читать: U-образность у Магнита = два типа товаров: стабильные (freq≈0)
    и «акционные» (freq≈1). Пятёрочка более равномерна.

05_freq_density_combined.png
  Тип: кривые плотности вероятности, обе сети на одном графике.
  Позволяет сравнить форму распределения: скошенность, вершины.

06_freq_hist_by_category.png
  Тип: гистограмма, facet по категории.
  Для каждой категории две гистограммы: Пятёрочка и Магнит наложены.
  Показывает, как форма распределения freq_effective варьируется по категориям.

07_freq_eff_vs_reg.png
  Тип: гистограмма, facet_grid(тип цены × сеть).
  Верхний ряд: эффективная цена. Нижний ряд: только регулярная цена.
  Как читать: сравните ряды — промо делает распределение более U-образным
    (акционные периоды концентрируются у freq≈1).

07b_price_delta_rub_effective.png
  Тип: гистограмма, facet по сети, шаг = 5 руб.
  Ось X: абсолютное изменение ЭФФЕКТИВНОЙ цены в рублях за неделю.
  Только периоды с фактическим изменением (changed_effective=TRUE).
  Выбросы >±200 руб. исключены.
  Как читать: форма распределения показывает типичный «масштаб» ценовых решений.
    Широкий разброс у Магнита = глубокие промо-скидки.

07c_price_delta_rub_regular.png
  Тип: гистограмма, facet по сети, шаг = 5 руб.
  Ось X: абсолютное изменение РЕГУЛЯРНОЙ цены в рублях.
  Только периоды с changed_regular=TRUE. Выбросы >±200 руб. исключены.
  Как читать: сравни с 07b — у Магнита регулярные цены меняются реже и
    на меньшие суммы, чем эффективные (основной движитель — промо).

07d_freq_regular_by_chain.png
  Тип: гистограмма, facet по сети.
  Ось X: freq_regular (доля недель с изменением РЕГУЛЯРНОЙ цены).
  Сравни с 04_freq_hist_by_chain.png: у Магнита freq_regular значительно ниже
    freq_effective — разрыв объясняется промо-активностью.

--- СКРИПТ 03: Корреляции ---

08_intercategory_cor_pyaterochka.png
09_intercategory_cor_magnit.png
10_intercategory_cor_combined.png
  Тип: тепловые карты (heatmaps), квадратные матрицы N×N.
  Цвет: красный = положительная корреляция, синий = отрицательная, белый = 0.
  Диагональ идёт слева-сверху вправо-вниз; диагональные элементы = 1.
  Числа внутри ячеек = коэффициент корреляции Пирсона.
  Как читать: равномерно красная матрица = централизованное ценообразование
    (все категории двигаются вместе). Пёстрая = децентрализованное.
  10_...combined: chg_rate усреднён по обеим сетям перед расчётом.

--- СКРИПТ 04: Декомпозиция дисперсии ---

11_variance_decomposition.png
  Тип: bar chart (горизонтальный).
  Ось X: R² adj. при единственном наборе FE (сеть / категория / магазин).
  Показывает, сколько % вариации дельта(эффективной цены) объясняется
  каждым структурным фактором по отдельности.

--- СКРИПТ 05: Кластеризация ---

12_silhouette_scores.png
  Тип: линейный график.
  Ось X: число кластеров K. Ось Y: средний silhouette score [-1,1].
  Вертикальная пунктирная линия = выбранное K.
  Как читать: чем выше score, тем лучше кластеры разделены.

13_cluster_pca.png
  Тип: PCA-биплот (первые 2 главных компонента признаков кластеризации).
  Каждая точка = категория, цвет = кластер, эллипс = выпуклая оболочка.
  Как читать: близкие точки = схожее ценовое поведение.

--- СКРИПТ 06: Панельная регрессия ---

14_coef_m1.png
  Тип: coef-plot с 95% ДИ.
  M1: два коэффициента — бета(Magnit) и бета(Promo) на одном графике.
  Ось X: оценка бета (дельта_ln эффективной цены). Красный = p<0.05.
  Как читать: если ДИ не пересекает 0 — эффект значим.

15_coef_magnit.png
  Только бета(Magnit) из M1. Систематическая разница между сетями в динамике
  эффективной цены при контроле категории и недели.

16_coef_promo.png
  бета(Promo) из трёх спецификаций (M1, M2, M3_pya) на одном графике.
  Устойчивость промо-эффекта к добавлению магазинных FE и к ограничению
  выборки только Пятёрочкой.

--- СКРИПТ 07: Проверки устойчивости ---

18_rc_magnit.png
  Только бета(Magnit): M1 / RC1 (регулярная цена) / RC2 (порог 0.5%) /
  RC3 (без выбросов). Если знак меняется в RC1 — разница между сетями
  объясняется главным образом промо-стратегией Магнита.

19_rc_promo.png
  Только бета(Promo): M1 / RC2 / RC3 (RC1 исключён — нет переменной промо).
  Устойчивость оценки промо-эффекта к выбору порога и удалению выбросов.

23_rf_importance.png
