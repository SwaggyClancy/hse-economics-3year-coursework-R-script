# Ценовая липкость в российском продуктовом ретейле

Курсовая работа по прикладной эконометрике — НИУ ВШЭ, 2026.  
**Авторы:** Абдуазизов, Андреасян, Андреев

Сравнительный анализ динамики цен в сетях **Пятёрочка** и **Магнит** методами панельной регрессии, декомпозиции дисперсии и кластеризации. **10 недель** наблюдений, **23 магазина** в Санкт-Петербурге, **496 390 записей**, **12 307 уникальных товаров**.

---

## Что исследуется

**Ценовая липкость (price stickiness)** — насколько редко и с какой амплитудой магазины меняют цены. Ключевые вопросы:

1. Есть ли систематическая разница в ценовых стратегиях Пятёрочки и Магнита?
2. Объясняется ли разница промо-акциями или регулярными ценами?
3. Что сильнее объясняет вариацию цен — сеть, категория товара или конкретный магазин?
4. На какие типы делятся категории по характеру ценовой динамики?
5. Есть ли Granger-причинность между частотой ценовых изменений двух сетей?

---

## Три концепции цены

| Переменная        | Описание                                              |
| ----------------- | ----------------------------------------------------- |
| `price_regular`   | Цена на полке (без акции)                             |
| `price_discount`  | Акционная цена (когда действует скидка)               |
| `effective_price` | Цена, которую платит покупатель = min(reg, discount)  |

Бинарный флаг `is_promo = TRUE` означает, что в этот период действует акция. Большинство метрик считается по **эффективной цене**, если не указано иное.

## Как запустить

### Требования

R ≥ 4.3.0. Пакеты устанавливаются автоматически при первом запуске `01_load_and_clean.R`.

Используемые пакеты: `tidyverse`, `readxl`, `fixest`, `gt`, `gtExtras`, `cluster`, `factoextra`, `ranger`, `scales`, `glue`, `ggrepel`.

### Запуск

```r
# Из RStudio: задать рабочую директорию и запустить главный скрипт
setwd("путь/к/папке/проекта")
source("run_all.R")

# Из терминала (из папки проекта):
Rscript run_all.R
```

Полный анализ выполняется за **~100 секунд** и создаёт все таблицы, графики и `output/report.html`.

### Входные данные

Файлы вида `combined_full_YYYY-MM-DD.xlsx` — в `data/raw/`. Каждый файл — снимок цен на одну неделю, лист `Combined`, колонки: `store_chain`, `store_code`, `category_id`, `date`, `product_id`, `name`, `price_regular`, `price_discount`, `is_available`.

## Описание скриптов

### 01 — Загрузка и очистка

Читает все Excel-файлы, унифицирует колонки, парсит цены (`"289,99"` → число), вычисляет:

- `effective_price = min(price_regular, price_discount)` при активной акции
- `is_promo`: флаг акции
- Лаговые цены (`lag_regular`, `lag_effective`) и процентные изменения
- `changed_effective` / `changed_regular`: бинарные флаги изменения > 1%
- `spell_length_weeks`: длина текущего ценового спелла (RLE по неизменным эффективным ценам)

### 02 — Описательные метрики

Считает метрики ценовой липкости для каждой пары (магазин, категория) и строит **графики 01–07d**:

- `freq_effective` / `freq_regular` — доля недель с изменением эффективной / регулярной цены
- `avg_spell_length` — средняя длина спелла
- `promo_share`, `avg_promo_depth` — акционная активность
- **07b/07c**: гистограммы абсолютных изменений цены в рублях (шаг 5 руб.)
- **07d**: частота изменений регулярной (полочной) цены

### 03 — Корреляции и синхронность

- **Кросс-цепочечная корреляция**: синхронны ли изменения Пятёрочки и Магнита в одной категории?
- **Межкатегорийная корреляция**: централизовано ли ценообразование внутри сети?
- **Тепловые карты (08–10)**: отдельно для Пятёрочки, Магнита и обеих вместе

Вывод: Пятёрочка — централизована (медиана межкат. корреляции ≈ 0.58), Магнит — децентрализован (≈ 0.13).

### 04 — Декомпозиция дисперсии

Последовательное добавление FE: цепочка → категория → магазин. Инкрементальный R² показывает «чистый» вклад каждого уровня. Категория объясняет больше всего.

### 05 — Кластеризация категорий

K-means по 5 признакам (freq_effective, avg_change, promo_share, volatility, avg_spell). Оптимальный K по silhouette с ограничением MIN_K=3. Визуализация — PCA-биплот.

### 06 — Панельная регрессия

Основная модель (M1):
```
Δln(P_eff) = β₁·Magnit + β₂·Promo + α_category + α_week + ε
```
Кластерные SE по магазину. Три спецификации: M1 (pooled), M2 (+ store FE), M3 (только Пятёрочка).

**Ключевой момент идентификации:** `category_id` уникален для каждой сети → `factor(category_id)` поглощает `is_magnit`. Решение: `cat_group` — кросс-цепочечные названия категорий (Молочный прилавок, Бакалея и т.д.).

HTML-таблица коэффициентов с подробными сносками: `output/tables/09_regression_coefs.html`.

### 07 — Проверки устойчивости

- **RC1** — только регулярные цены (без промо): проверяем, не объясняется ли эффект Магнита одними акциями
- **RC2** — порог 0.5% вместо 1%: чувствительность к выбору порога
- **RC3** — исключение выбросов |ΔP| > 50%: влияние экстремальных наблюдений

### 08 — Регрессии на частоту изменений

Дополнительные модели с зависимой `changed_effective` (0/1):

- **LPM**: линейная модель вероятности — β(Промо) и β(Магнит) в п.п.
- **Granger**: предсказывает ли лаг одной сети частоту другой?
- **AR(1)**: ρ < 0 = ценовое торможение; ρ > 0 = инерция изменений
- **Random Forest**: важность групп переменных (категория/неделя/магазин/промо/сеть)

## Ключевые результаты

|                               | Пятёрочка | Магнит    |
| ----------------------------- | --------- | --------- |
| Частота изменений (рег. цены) | ~27%      | ~16%      |
| Частота изменений (эфф. цены) | ~32%      | ~21%      |
| Доля промо-периодов           | ~13%      | **~38%**  |
| Глубина скидки                | ~20%      | ~25%      |
| Средний спелл, нед            | ~4.2      | ~5.3      |
| Межкат. корреляция (медиана)  | ~0.58     | ~0.13     |

**Панельная регрессия (M1):**

| Переменная   | β       | Интерпретация                                        |
| ------------ | ------- | ---------------------------------------------------- |
| `is_magnit`  | −0.009  | Магнит меняет цены на ~0.9 п.п. реже/меньше, чем Пятёрочка |
| `is_promo_i` | −0.084  | Акция снижает эффективную цену на ~8.4% (log)        |

**Вывод из RC1:** при анализе только регулярных цен знак при Магните меняется на положительный. **Ценовое преимущество Магнита полностью обусловлено промо-акциями** — без скидок его полочные цены выше.

**Две ценовые стратегии:**
- **Пятёрочка** — гибкое ценообразование: полочные цены часто корректируются, промо умеренные.
- **Магнит** — Hi-Lo стратегия: полочные цены стабильны, агрессивные и глубокие промо создают видимость низких цен.
