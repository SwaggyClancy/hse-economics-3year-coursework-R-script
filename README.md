# Ценовая липкость в российском продуктовом ретейле

Курсовая работа по прикладной эконометрике — НИУ ВШЭ, 2026.
**Авторы:** Абдуазизов, Андреасян, Андреев

Сравнительный анализ динамики цен в сетях **Пятёрочка** и **Магнит** методами панельной регрессии, декомпозиции дисперсии и кластеризации. 9 недель наблюдений, 23 магазина в Санкт-Петербурге, 447 162 записей, 12 186 уникальных товаров.

---

## Что исследуется

**Ценовая липкость (price stickiness)** — насколько редко и с какой амплитудой магазины меняют цены. Основные вопросы:

1. Есть ли систематическая разница в ценовых стратегиях Пятёрочки и Магнита?
2. Объясняется ли разница промо-акциями или регулярными ценами?
3. Что сильнее объясняет вариацию цен — сеть, категория товара или конкретный магазин?
4. На какие типы делятся категории по характеру ценовой динамики?

## Как запустить

### Требования
R ≥ 4.3.0. Пакеты устанавливаются автоматически при первом запуске скрипта 01.

Используемые пакеты: `tidyverse`, `readxl`, `fixest`, `gt`, `gtExtras`, `cluster`, `factoextra`, `scales`, `glue`, `ggrepel`.

### Запуск

```r
# Из RStudio: задать рабочую директорию и запустить главный скрипт
setwd("путь/к/папке/проекта")
source("run_all.R")

# Из терминала (из папки проекта):
Rscript run_all.R
```

Полный анализ выполняется за ~30 секунд и создаёт все таблицы и графики в `output/`.

### Входные данные

Файлы вида `combined_full_YYYY-MM-DD.xlsx` кладутся в `data/raw/`. Каждый файл — снимок цен на одну неделю, лист `Combined`, колонки: `store_chain`, `store_code`, `category_id`, `date`, `product_id`, `name`, `price_regular`, `price_discount`, `is_available`.

## Описание скриптов

### 01 — Загрузка и очистка

Читает все Excel-файлы, унифицирует названия колонок, парсит цены (формат `"289,99"` → число), вычисляет:
- `effective_price` = цена с учётом скидки (если `price_discount < price_regular`)
- `is_promo` = флаг активной акции
- лаговые цены и процентные изменения (`delta_effective_pct`, `delta_regular_pct`)
- длину ценового спелла (RLE — число подряд идущих недель без изменения цены)

Сохраняет очищенную панель в `data/processed/panel.rds`.

### 02 — Описательные метрики

Считает метрики ценовой липкости для каждой пары (магазин, категория):
- `freq_effective` — доля недель с `|ΔP_eff| > 1%`
- `avg_spell_length` — средняя длина спелла (недель)
- `promo_share` — доля периодов с акцией
- `avg_promo_depth` — средняя глубина скидки

Агрегирует по сети (взвешенное среднее) и строит три графика: частота изменений по категориям, промо vs. частота (scatter), распределение длин спеллов.

### 03 — Корреляции и синхронность

**Гипотеза о внешних шоках:** если сети реагируют на одинаковые стимулы (инфляция, сезонность), недельная частота изменений должна коррелировать между ними.

**Гипотеза о централизации:** если ценообразование централизовано, все категории внутри одной сети меняют цены синхронно → высокие межкатегорийные корреляции.

Результат: Пятёрочка — медиана межкатегорийной корреляции 0.72 (централизована), Магнит — 0.12 (децентрализован).

### 04 — Декомпозиция дисперсии

Методом последовательного добавления фиксированных эффектов (chain → category → store) оценивает, какой уровень объясняет больше вариации цен. Использует `fixest::feols` для скорости.

### 05 — Кластеризация категорий

K-means по пяти признакам (`freq_effective`, `avg_change_effective`, `promo_share`, `volatility`, `avg_spell`). Оптимальное число кластеров выбирается по максимуму среднего silhouette score. Визуализация — PCA-биплот.

### 06 — Панельная регрессия

Основная модель:

```
Δln(P_eff) = β₁·Magnit + β₂·Promo + α_category + α_week + ε
```

Кластерные SE по магазину. Три спецификации:
- **M1** — pooled: category + week FE
- **M2** — тройные FE: category + week + store (поглощает chain-эффект, Magnit убран)
- **M3** — только Пятёрочка: category + week FE

**Ключевой момент идентификации:** `category_id` различается между сетями, поэтому `factor(category_id)` как FE поглощает `is_magnit` (perfect collinearity). Решение: использовать `cat_group` — кросс-цепочечные метки категорий (Молочный прилавок, Бакалея и т.д.), которые есть в обеих сетях.

### 07 — Проверки устойчивости

Три альтернативные спецификации для проверки робастности M1:
- **RC1** — только регулярные цены (без промо)
- **RC2** — порог изменения 0.5% вместо 1%
- **RC3** — исключение выбросов `|ΔP| > 50%`

## Ключевые результаты

| | Пятёрочка | Магнит |
|---|---|---|
| Частота изменений (рег. цены) | 27.0% | 15.8% |
| Частота изменений (эфф. цены) | 32.4% | 20.9% |
| Доля промо-периодов | 13.2% | **37.7%** |
| Глубина скидки | 20.2% | 25.1% |
| Средний спелл, нед | 4.2 | 5.3 |
| Межкат. корреляция (медиана) | 0.72 | 0.12 |

**Панельная регрессия (M1):**

| Переменная | β | SE | p |
|---|---|---|---|
| `is_magnit` | −0.0087 | 0.0010 | \*\*\* |
| `is_promo_i` | −0.0836 | 0.0028 | \*\*\* |

**Вывод из robustness RC1:** при анализе только регулярных цен знак при Магните меняется на положительный (+0.0036\*\*\*). Это значит, что **ценовое преимущество Магнита полностью обусловлено промо-акциями** — без скидок его регулярные цены выше, чем у Пятёрочки.

**Две ценовые стратегии:**
- **Пятёрочка** — гибкое ценообразование: регулярные цены часто меняются, промо умеренные.
- **Магнит** — Hi-Lo стратегия: регулярные цены стабильны, но агрессивные и частые промо создают видимость низких цен.
