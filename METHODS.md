## 1. Данные

### 1.1 Источник и сбор данных

Данные собраны вручную с официальных мобильных приложений и сайтов розничных сетей
«Пятёрочка» и «Магнит» в период с 26 марта по 21 мая 2026 года. Наблюдения проводились
еженедельно, в один и тот же день недели, что обеспечивает сопоставимость периодов.

Итоговая выборка охватывает:

- **23 магазина** в Санкт-Петербурге (11 магазинов Пятёрочки и 12 магазинов Магнита)
  в трёх районах: Центральный, Петроградский, Невский
- **10 еженедельных волн** наблюдений
- **496 390 наблюдений** (пара «товар — неделя»)
- **12 307 уникальных товаров** в **15 товарных категориях**

Для каждого товара в каждом магазине и каждую неделю фиксировались: регулярная цена
(`price_regular`), акционная цена (`price_discount`), наличие товара на полке,
идентификатор категории.

### 1.2 Ограничения выборки

Выборка магазинов не является случайной: магазины выбирались исходя из географической
доступности и сопоставимости районов. Оба ретейлера относятся к формату «магазин у дома»
(proximity store), поэтому результаты не распространяются на гипермаркеты и
супермаркеты других форматов.

## 2. Подготовка данных

### 2.1 Построение панели

Исходные данные собраны в формате Excel (10 файлов по одному на неделю). На первом этапе
все файлы объединяются в единую длинную панель. Строковые значения цен (формат `«289,99»`)
преобразуются в числовой формат с заменой запятой на точку.

Особенность данных Магнита: в исходных таблицах колонки `price_regular` и `price_discount`
переставлены местами по сравнению с Пятёрочкой. Для устранения этой инверсии применяется
автоматическая проверка: если `price_discount > price_regular`, колонки меняются местами.

После объединения удаляются дубли по ключу (сеть, магазин, товар, дата).

### 2.2 Эффективная цена и флаг акции

Для каждого наблюдения вычисляется **эффективная цена** — цена, которую фактически
платит покупатель:

```
P_eff = price_discount,  если price_discount < price_regular и price_discount > 0
P_eff = price_regular,   иначе
```

Флаг `is_promo = TRUE`, если товар находится в акции (P_eff < P_reg).

### 2.3 Лаги, изменения цен и пороговый фильтр

Для каждой тройки (сеть, магазин, товар) данные сортируются по дате, после чего
вычисляются:

- лаговые цены: `lag_regular`, `lag_effective`
- относительные изменения цен:
  ```
  delta_effective_pct = (P_eff_t − P_eff_{t−1}) / P_eff_{t−1}
  delta_regular_pct   = (P_reg_t − P_reg_{t−1}) / P_reg_{t−1}
  ```
- бинарные индикаторы значимого изменения:
  ```
  changed_effective = |delta_effective_pct| > 1%
  changed_regular   = |delta_regular_pct|   > 1%
  ```

Порог в 1% позволяет отфильтровать округления и технические флуктуации цен, не являющиеся
реальными ценовыми решениями.

### 2.4 Ценовые спеллы

**Ценовой спелл** — период последовательных недель без изменения эффективной цены.
Спеллы вычисляются алгоритмом run-length encoding (RLE): при каждом изменении цены
начинается новый спелл, ему присваивается идентификатор `spell_id = cumsum(changed_effective)`.
Длина спелла `spell_length_weeks` — количество наблюдений с одинаковым `spell_id`.

Длина ценового спелла является обратным показателем ценовой гибкости: чем длиннее
средний спелл, тем более «липкими» являются цены.

## 3. Описательные метрики ценовой липкости

Для каждой тройки (магазин, категория) рассчитывается набор метрик.

**Частота изменений** (freq_effective) — доля наблюдений с `changed_effective = TRUE`:

```
freq_effective = #{t : |ΔP_eff,t| > 1%} / #{t : lag_effective ≠ NA}
```

**Доля акционных периодов** (promo_share) — доля наблюдений с `is_promo = TRUE`:

```
promo_share = #{t : is_promo = TRUE} / #{всех наблюдений}
```

**Средняя глубина скидки** (avg_promo_depth) — средняя относительная скидка
в акционные периоды:

```
avg_promo_depth = mean[(P_reg − P_eff) / P_reg | is_promo = TRUE]
```

**Средний размер изменения** (avg_change_effective) — среднее абсолютное
процентное изменение в периоды с `changed_effective = TRUE`.

**Средняя длина спелла** (avg_spell_length) — среднее число недель без изменения
эффективной цены.

Все метрики агрегируются по сети через взвешенное среднее (веса = число наблюдений
в данной категории-магазине).

## 4. Анализ синхронности ценовых изменений

### 4.1 Кросс-цепочечная синхронность

Для проверки гипотезы о том, что сети реагируют на общие внешние шоки (инфляция,
сезонность, изменения издержек), вычисляется корреляция Пирсона еженедельной частоты
изменений между Пятёрочкой и Магнитом по каждой категории:

```
r(category) = Corr(chg_rate_Pyaterochka_t, chg_rate_Magnit_t)  по неделям t
```

где `chg_rate` — средняя доля товаров в данной (сеть, категория, неделя) с изменением
цены. Высокая положительная корреляция свидетельствует о синхронном поведении сетей
(доминирование внешних факторов). Нулевая или отрицательная — о независимом принятии
ценовых решений.

### 4.2 Межкатегорийная синхронность (показатель централизации)

Централизованность ценообразования внутри сети оценивается через попарные корреляции
временных рядов `chg_rate` между категориями:

```
r(cat_i, cat_j) = Corr(chg_rate_i_t, chg_rate_j_t)  по неделям t
```

Если ценообразование централизовано (единый центр принятия решений), все категории
меняют цены одновременно → высокие положительные попарные корреляции.
Если децентрализовано — корреляции близки к нулю или отрицательны.

## 5. Декомпозиция дисперсии

Методом последовательного добавления фиксированных эффектов (FE) оценивается
относительный вклад трёх структурных факторов в вариацию еженедельных изменений
эффективной цены:

| Модель | FE                         | R² adj. |
| ------ | -------------------------- | ------- |
| 1      | Сеть                       | r₁      |
| 2      | Категория                  | r₂      |
| 3      | Магазин                    | r₃      |
| 4      | Сеть + Категория           | r₄      |
| 5      | Сеть + Категория + Магазин | r₅      |

Инкрементальный вклад каждого фактора: ΔR²(магазин) = r₅ − r₄.

Оценка проводится через `fixest::feols` (R-пакет) для эффективной работы с большим
числом FE без явного формирования матрицы дамми-переменных.

## 6. Кластеризация товарных категорий

### 6.1 Признаки и предобработка

Для каждой категории (объединяя обе сети через взвешенное среднее) формируется
вектор из пяти признаков:

1. `freq_effective` — частота изменений эффективной цены
2. `avg_change_effective` — средний размер изменения
3. `promo_share` — доля акционных периодов
4. `volatility_effective` — стандартное отклонение delta_effective_pct
5. `avg_spell_length` — средняя длина ценового спелла

Перед кластеризацией признаки стандартизируются (z-score) через `scale()`, чтобы
исключить влияние разных единиц измерения.

### 6.2 Выбор числа кластеров и содержательное ограничение

Оптимальное число кластеров K выбирается по максимуму **среднего silhouette score**:

```
s(i) = (b(i) − a(i)) / max(a(i), b(i))
```

где `a(i)` — среднее внутрикластерное расстояние для объекта i,
`b(i)` — среднее расстояние до ближайшего чужого кластера.
Silhouette score лежит в [-1, 1]; чем выше, тем лучше разделены кластеры.
Перебирается K от 2 до N_categories − 1.

При малом числе категорий (8–9) силуэт математически склонен возвращать K=2:
сбалансированное разбиение 4/4 даёт высокий score даже без содержательного смысла.
Поэтому применяется **содержательное ограничение** `MIN_K = 3`: если силуэт предлагает
K < MIN_K, принудительно используется K = MIN_K, о чём выводится предупреждение.
Это исследовательское решение, обоснованное необходимостью содержательной типологии
(«акционные» / «стабильные» / «смешанные» категории).

### 6.3 Финальная кластеризация

Применяется алгоритм K-means с параметрами: `set.seed(42)` (воспроизводимость),
`nstart = 50` (50 случайных начальных разбиений, берётся лучшее), `iter.max = 200`.

Для визуализации строится PCA-биплот: первые два главных компонента объясняют
наибольшую долю вариации признаков, на них проецируются категории с
цветовой разметкой кластеров.

## 7. Панельная регрессия

### 7.1 Основная спецификация (M1)

Для оценки систематической разницы в ценовой динамике между сетями оценивается
панельная регрессия:

```
Δln(P_eff)_{i,t} = β₁ · Magnit_i + β₂ · Promo_{i,t} + α_c + α_t + ε_{i,t}
```

где:

- `Δln(P_eff)_{i,t} = ln(P_eff_{i,t}) − ln(P_eff_{i,t−1})` — логарифмическое
  изменение эффективной цены товара i в неделю t
- `Magnit_i = 1` если наблюдение принадлежит сети Магнит, иначе 0
- `Promo_{i,t} = 1` если товар i в неделю t находится в акции
- `α_c` — фиксированные эффекты по категории товара (контролируют корзину)
- `α_t` — фиксированные эффекты по неделе (контролируют общие ценовые шоки)
- `ε_{i,t}` — идиосинкратическая ошибка

Стандартные ошибки кластеризованы на уровне магазина (cluster-robust SE)
для учёта корреляции ошибок внутри магазина во времени.

### 7.2 Ключевой вопрос идентификации

Идентификатор категории (`category_id`) различается между сетями: у Пятёрочки
коды вида `251C1xxxx`, у Магнита — `6xxxx` / `4xxxx`. Если использовать
`factor(category_id)` как FE, каждый идентификатор принадлежит ровно одной сети,
и переменная `Magnit` поглощается этими FE (perfect collinearity), становясь
неидентифицируемой.

Решение: используются **кросс-цепочечные категориальные группы** (`cat_group`) —
единые человекочитаемые названия (Молочный прилавок, Бакалея и др.), присутствующие
в обеих сетях. Это обеспечивает вариацию `Magnit` внутри каждой группы FE,
делая оценку β₁ возможной.

### 7.3 Дополнительные спецификации

**M2** — тройные FE (категория + неделя + магазин):

```
Δln(P_eff) = β₂ · Promo + α_c + α_t + α_store + ε
```

Фиксированный эффект магазина поглощает chain-эффект, поэтому `Magnit` из M2
исключён. M2 проверяет устойчивость коэффициента при `Promo` при полном контроле
на магазинную гетерогенность.

**M3** — регрессия только по Пятёрочке:

```
Δln(P_eff) = β₂ · Promo + α_c + α_t + ε  [только Pyaterochka]
```

Позволяет сравнить эффект промо отдельно по сетям. Аналогичная регрессия по
Магниту (m3_mag) неидентифицируема: промо-акции Магнита синхронизированы на уровне
сети — внутри каждой пары (категория × неделя) переменная `Promo` константна,
что исключает её идентификацию при наличии category×week FE.

### 7.4 Техническая реализация

Все регрессии оцениваются через R-пакет `fixest` (функция `feols`), который
реализует алгоритм итерируемых проекций Кивета–Хирсбруннера для эффективной
оценки моделей с несколькими наборами FE без явного формирования dummy-матриц.

## 8. Проверки устойчивости

Для проверки робастности результатов M1 оцениваются три альтернативные спецификации.

**RC1 — Только регулярные цены.**
Зависимая переменная заменяется на `Δln(P_reg)` — изменение регулярной (не акционной)
цены. Переменная `Promo` из модели исключается. Если β(Magnit) сохраняет знак
и значимость, разница между сетями не объясняется исключительно промо-стратегиями.

**RC2 — Альтернативный порог изменения (0,5%).**
Порог `changed_effective` снижается с 1% до 0,5%. В выборку включаются более мелкие
ценовые изменения. Проверяется чувствительность результата к выбору минимального
значимого изменения.

**RC3 — Исключение выбросов.**
Из выборки удаляются наблюдения с `|Δln(P_eff)| > 50%` (экстремальные изменения,
возможно обусловленные ошибками данных или уценкой при списании). Проверяется,
не искажают ли outliers основные выводы.

Во всех трёх случаях используются те же FE (cat_group + week) и кластеризация SE
по магазину, что и в M1. Результаты сравниваются по знаку, величине и
статистической значимости коэффициентов.

## 9. Регрессии на частоту изменений цен (скрипт 08)

Дополнительно к основной регрессии (раздел 7) оцениваются четыре типа моделей,
в которых зависимой переменной является **вероятность изменения эффективной цены**
(`changed_effective`, бинарная 0/1) либо её агрегаты по сети.

### 9.1 Топ-5 корреляций по категориям

Для каждой категории из попарных корреляций (раздел 4.2) извлекается топ-5
наиболее скоррелированных пар. Это позволяет быстро идентифицировать «товарных
соседей» — категории, ценовые решения которых движутся синхронно. Результат
публикуется как `12_top5_correlations.csv/.html`.

### 9.2 Линейная модель вероятности (LPM)

Оцениваются три спецификации методом OLS (LPM — Linear Probability Model):

```
changed_effective_{i,t} = β₁·Magnit_i + β₂·Promo_{i,t} + α_c + α_t + ε
```

Коэффициент β интерпретируется как изменение **вероятности** изменения цены в
процентных пунктах. LPM выбран вместо Probit/Logit из-за простой интерпретации
коэффициентов и совместимости с FE-оценкой через `feols`.

Предпосылка для LPM: необходимо убедиться, что предсказанные значения не выходят
за [0, 1] массово. При наличии FE это выполняется благодаря within-вариации.

### 9.3 Кросс-цепочечная синхронность: Granger-подход

Оцениваются «Granger-регрессии» — проверяется, предсказывает ли прошлая частота
изменений одной сети текущую частоту другой:

```
chg_rate_Pya_t = β₁·chg_rate_Pya_{t-1} + β₂·chg_rate_Mag_{t-1} + α_category + ε
chg_rate_Mag_t = β₁·chg_rate_Mag_{t-1} + β₂·chg_rate_Pya_{t-1} + α_category + ε
```

Значимость лага «чужой» сети (β₂) интерпретируется как свидетельство Granger-
причинности. Это не строгий тест (без F-теста на исключение), но показывает
направление и силу кросс-лагового эффекта. FE по категории убирают категорийные
фиксированные уровни частоты.

### 9.4 AR(1) панельная модель

Оцениваются три AR(1)-спецификации на уровне товара:

```
changed_effective_{i,t} = ρ·changed_effective_{i,t-1} [+ β₂·Promo + β₃·Magnit] + α_c + α_t + ε
```

Лаг DV `changed_effective_{t-1}` вычисляется на уровне тройки (сеть, магазин, товар).
Знак ρ интерпретируется так:
- ρ < 0: «ценовое торможение» — после изменения цена с повышенной вероятностью
  остаётся на месте (menu cost logic);
- ρ > 0: «инерция» — серия изменений следует одно за другим (seasonal adjustment).

### 9.5 Random Forest: важность переменных

Обучается ансамбль деревьев (`ranger`, 300 деревьев, `importance = "impurity"`)
для предсказания `changed_effective` из набора признаков:
`is_magnit`, `is_promo_i`, `cat_fct`, `week_fct`, `store_fct`.

Важность агрегируется по группам переменных (FE-группы vs субстантивные
предикторы), чтобы получить интерпретируемый рейтинг. FE-переменные (категория,
неделя, магазин) как правило доминируют по важности — это подтверждает, что
структурные факторы объясняют большую часть вероятности изменения цены.
